The three roles

  1. Planner

    Owns the plan and decides the approach. Does not edit code. Communicates the plan to the Executor via chat and adjusts strategy based on what the Verifier reports.

  2. Executor

    The only role that can edit code and run commands in the terminal. Implements the Planner's instructions and responds to fix-requests from the Verifier.

  3. Verifier

    Reviews the Executor's code, runs tests, and decides whether the solution is correct. Can send the work back to the Executor for fixes until the criteria are met.
